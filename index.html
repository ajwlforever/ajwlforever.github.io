Hexo的认识-1

在知道有hexo这个东西前，由于对github和echarts的使用，我电脑中的git、node.js、npm（node.js已经集成了npm）的安装已经全部完成。

node.js安装：https://blog.csdn.net/weixin_41585557/article/details/79155526
//npm config set cache “D:\Program Files\Nodejs\node_global” //配置npm的全局模块的存放路径以及cache的路径。
//npm config set prefix “D:\Program Files\Nodejs\node_cache”
二、
在b站的视频中安装hexo是在cmd 用

npm install hexo-cli -g. //-g这里是全局的意思。npm install 本地安装与全局安装的区别：https://www.cnblogs.com/qiangxia/p/5264720.html

我安装的是

npm install hexo -g

hexo-cli与hexo的关系 http://www.one-more-tech.info/Hexo-%E4%B8%8E-Hexo-cli-%E7%9A%84%E5%85%B3%E7%B3%BB%E3%80%81%E5%85%A5%E9%97%A8%E6%95%99%E7%A8%8B/index/

这时可以用 hexo -v 查看hexo版本 在cmd和git中操作都可以，我是选择在git中操作。
ps：这时有可能会说hexo不是命令或不是外部命令，这是因为系统环境变量加到path之后
C:\Users\ajwlf\node_modules\hexo\node_modules.bin 这里的.bin里面只有两个文件，千万不要去找install到npm里的hexo的.bin，一定是在c盘。
这里我耽误了好长时间，就是因为没找到。。。甚至一度以为时出了别的问题，重新下载了好几遍hexo和hexo-cli

三、安装hexo
安装完之后对博客文件（随便建一个空文件即可）夹进行gitbash 将hexo安装到博客文件夹

hexo init
